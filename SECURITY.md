# Security

Please report security issues responsibly.

## What To Report

- Vulnerabilities that expose stored Driftwatch data.
- Bugs that could enable abuse, unauthorized actions, or unsafe automation.
- Permission handling flaws.
- Issues that could leak bot tokens or credentials.
- Any behavior that contradicts Driftwatch's defensive-use boundaries.

## What To Include

- A clear description of the issue.
- Steps to reproduce, if safe to share privately.
- Expected and actual behavior.
- Affected version or commit.
- Relevant logs with secrets removed.

Do not publicly disclose security issues before maintainers have had a reasonable chance to investigate and respond. Do not include live tokens, passwords, private server data, or exploit instructions in public reports.
