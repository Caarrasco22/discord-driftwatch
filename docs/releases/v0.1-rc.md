# Driftwatch v0.1 Release Candidate

Driftwatch v0.1 RC is prepared for controlled testing by authorized Discord server owners and administrators.

Core purpose:

> Review your server, set an accepted reference, and track important changes over time.

## What v0.1 RC Does

- Reviews visible current configuration risks with `/driftwatch check`.
- Stores an accepted baseline reference with `/driftwatch baseline action:create`.
- Compares future drift with `/driftwatch baseline action:compare`.
- Reviews recent administrative activity from Discord audit logs with `/driftwatch logs`.
- Generates practical stored reports for current risk, baseline comparison, logs, and latest runs.
- Stores local SQLite records for summaries, findings, skipped checks, baselines, and report references.

## Main Commands

```text
/driftwatch help
/driftwatch data
/driftwatch setup
/driftwatch check
/driftwatch logs
/driftwatch baseline action:create
/driftwatch baseline action:list
/driftwatch baseline action:compare
/driftwatch report source:latest
/driftwatch report source:current-risk
/driftwatch report source:baseline-compare
/driftwatch report source:logs
/driftwatch delete-data confirm:false
```

`/driftwatch impact` is present but experimental and should not be treated as a complete diagnostic.

## Recommended Usage Flow

1. Run `/driftwatch setup`.
2. Run `/driftwatch check`.
3. Run `/driftwatch logs`.
4. Manually review and fix what matters.
5. When the server is in an accepted state, run `/driftwatch baseline action:create`.
6. Use `/driftwatch baseline action:compare` in future reviews.

A baseline is only a stored reference/snapshot of an accepted state. It is not proof that a server is secure.

## Data Stored

- Guild settings and authorized role IDs.
- Sanitized baseline snapshots.
- Audit run summaries.
- Derived finding summaries.
- Skipped checks.
- Report message references.
- Timestamps and retention settings.

## Data Not Stored

- Message content.
- DMs.
- User tokens or bot tokens.
- Passwords.
- Emails.
- IP addresses.
- Raw audit logs.
- Full Discord object dumps.
- Unnecessary personal data.

## Required Permissions

Minimum bot permissions:

- View Channels.
- Send Messages.
- Embed Links.
- View Audit Log.

Administrator is not required by default. Missing optional permissions should become skipped or limited checks, not hard crashes.

## Safety Model

- No server mutation.
- No auto-fix.
- `safeToAutoFix` remains false.
- No privileged intents in v0.1/basic mode.
- No message reading.
- No user tokens or selfbot behavior.
- No raw audit log storage.
- Uses official Discord bot APIs for authorized defensive review.

## Known Limitations

- No auto-fix.
- No message reading.
- No member-level deep audit.
- No Guild Members Intent.
- Audit logs depend on Discord availability and retention.
- A baseline is only an accepted reference, not a security guarantee.
- `/driftwatch impact` is experimental.
- Self-hosting requires some technical setup.
- Current checks are heuristic and intended to support manual review.

## Tester Checklist Summary

- Install dependencies.
- Configure `.env`.
- Run `npm run doctor`.
- Run `npm run validate`.
- Deploy slash commands.
- Start the bot.
- Test the recommended Discord command flow in a private server.
- Confirm missing permissions produce skipped or limited checks.
- Confirm `/driftwatch delete-data confirm:false` does not delete data.

## Feedback

Report feedback with:

- Hosting environment.
- Command used.
- Expected result.
- Actual result.
- Screenshot or sanitized logs when useful.

Do not share bot tokens, private server data, raw audit logs, message content, or personal data in reports.
